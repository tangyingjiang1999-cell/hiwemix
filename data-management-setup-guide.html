1
在 Supabase 执行建表 SQL

打开 Supabase Dashboard → SQL Editor → New query

复制 supabase-formula-schema.sql 文件末尾新增的 guide 表 SQL（从 -- Application Guide 表 注释开始到 NOTIFY pgrst, 'reload schema'），粘贴执行。

这会创建 guide_categories 和 guides 两张表（含触发器、RLS 公开读策略）。

✓ 执行成功后，/api/guides 公开读路由即可工作（返回空数组）。
2
迁移指南数据到 Supabase

用 admin 账号（admin/admin123）登录网站后，浏览器直接访问：

http://localhost:3000/api/admin/migrate-guides

或线上：https://hiwemix.com/api/admin/migrate-guides

该路由会把 guide-data.ts 的 5 个分类 + 5 篇指南用 service_role upsert 到 Supabase。

✓ 看到 {"success":true,"migrated":{"categories":5,"guides":5}} 即成功。
3
删除临时迁移路由

迁移完成后，删除整个目录：

src/app/api/admin/migrate-guides/

这是一次性迁移工具，不应留在生产代码中。

4
本地验证（可选）

本地 dev server 因 16GB 内存不足在 Turbopack 首次编译时可能 OOM。如需本地验证，增大 Node 内存：

NODE_OPTIONS=--max-old-space-size=4096 npm run dev

然后访问 /admin/data（需 admin 登录），或从任意业务页面底部点「Data Management」按钮进入。

⚠ 线上 Vercel 无此内存问题（serverless 按需分配），本地才需调大内存。
5
部署到 Vercel

确认 3 个 Supabase 环境变量已在 Vercel Production 配置（之前已添加）：

  • NEXT_PUBLIC_SUPABASE_URL
  • NEXT_PUBLIC_SUPABASE_ANON_KEY
  • SUPABASE_SERVICE_ROLE_KEY（仅服务端）

提交代码并推送，Vercel 自动部署。部署后 hiwemix.com 即生效。
