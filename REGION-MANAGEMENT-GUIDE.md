## 1. 数据库迁移（必须先执行）

### 方法 1：通过 Supabase Dashboard 执行
1. 登录 Supabase Dashboard
2. 进入 SQL Editor
3. 打开文件 `supabase-migrations/20260716000000_add_regions_table.sql`
4. 复制并执行 SQL 脚本
5. 验证执行结果

### 方法 2：通过 Supabase CLI 执行（推荐）
```bash
# 在项目根目录执行
npx supabase migration up
```

### 迁移内容
脚本将创建：
- **regions 表**：存储产地代码（JPN、EUR、USA、CHN、KOR）
- **修改 brands 表**：将 region 字段从 CHECK 约束改为外键关联到 regions 表

## 2. 新增的功能

### 2.1 管理员后台 - 品牌管理面板

**新增产地功能**：
- 在品牌管理页面，点击右上角的 "+ 新增产地" 按钮
- 在弹出的对话框中输入产地代码（如 SEA 代表东南亚）
- 产地代码会自动转为大写，最长 10 个字符
- 新增的产地立即出现在所有品牌的选择器中

**删除产地功能**：
- 在品牌管理页面底部，显示所有已有的产地
- 每个产地旁边有一个删除按钮（红色）
- 点击删除时，如果该产地下有关联品牌，系统会提示错误
- 如果没有关联品牌，则成功删除

**新增/编辑品牌时选择产地**：
- 打开新增品牌或编辑品牌对话框
- 产地下拉框现在显示所有从数据库中获取的产地代码
- 选择一个产地后保存即可

### 2.2 搜索面板 - 产地筛选

**新增产地下拉框**：
- 在搜索面板中，新增了一个"产地"选择框（第一行第一列）
- 默认显示"全部产地"
- 可以选择一个产地（如 JPN）来筛选颜色

**产地筛选逻辑**：
1. 选择产地（如 JPN）
2. 系统先找出所有该产地的品牌（如 Toyota、Honda、Nissan）
3. 再在这些品牌的颜色中进行搜索
4. 与其他筛选条件（品牌、颜色代码、颜色名称等）组合使用

**重置功能**：
- 点击"重置"按钮会同时清除产地筛选器
- 所有筛选条件都会被重置

## 3. 使用示例

### 示例 1：新增一个新产地
1. 登录管理员后台
2. 进入"品牌管理"标签
3. 点击"+ 新增产地"按钮
4. 输入"SEA"（东南亚）

2. 在"产地"下拉框中选择"JPN"
3. 点击"搜索"
4. 结果只显示日本品牌（Toyota、Honda、Nissan 等）的颜色
5. 可以再结合其他筛选条件（如颜色类型=Metallic）进一步缩小范围

### 示例 3：删除一个产地
1. 登录管理员后台
2. 进入"品牌管理"标签
3. 在底部的"已有产地"区域找到要删除的产地
4. 点击红色的删除按钮
5. 如果该产地下有关联品牌，系统会提示"该产地下还有关联的品牌，无法删除"
6. 如果没有关联品牌，则成功删除

## 4. 技术细节

### 4.1 数据库结构

**regions 表**：
```sql
CREATE TABLE regions (
  code TEXT PRIMARY KEY,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

**brands 表的 region 字段**：
- 类型：TEXT
- 外键关联到 regions(code)
- ON UPDATE CASCADE（产地代码修改时，所有关联品牌自动更新）
- ON DELETE RESTRICT（删除产地时，如果有关联品牌则阻止删除）

### 4.2 API 端点

**公开 API（无需认证）**：
- `GET /api/regions` - 获取所有产地列表

**管理员 API（需要认证）**：
- `GET /api/admin/regions` - 获取所有产地
- `POST /api/admin/regions` - 新增产地
- `DELETE /api/admin/regions` - 删除产地

### 4.3 前端组件

**BrandsPanel 组件**：
- 新增了"新增产地"按钮和对话框
- 从 API 获取产地列表
- 显示并管理已有产地的列表
- 删除产地时检查关联品牌

**SearchPanel 组件**：
- 新增了产地下拉框
- 从 API 获取产地列表
- 产地选择会触发搜索条件更新

**page.tsx 搜索逻辑**：
- 在 `handleSearch()` 中新增产地筛选
- 先按产地筛选品牌，再按品牌筛选颜色
- 支持与其他筛选条件组合使用

## 5. 注意事项

### 5.1 权限控制
- 只有管理员可以新增、编辑、删除产地
- 公开的 API 端点（`GET /api/regions`）无需认证
- 前端会根据用户角色隐藏或显示管理功能

### 5.2 数据完整性
- 删除产地时，系统会检查是否有关联品牌
- 如果有关联品牌，删除操作会被阻止
- 需要先删除或修改所有使用该产地的品牌，才能删除产地

### 5.3 向后兼容
- 现有品牌数据不受影响
- 现有品牌的所有 region 值（JPN、EUR、USA、CHN、KOR）会自动迁移到新的 regions 表
- 现有的搜索功能保持不变

### 5.4 国际化
- 产地代码保持大写英文缩写（如 JPN、EUR）
- 产地代码便于排序和搜索
- 如需显示产地全称，可以在前端维护一个映射表

## 6. 故障排除

### 问题 1：无法新增产地
**原因**：可能没有管理员权限
**解决**：确保使用管理员账号登录

### 问题 2：删除产地时提示"该产地下还有关联的品牌"
**原因**：该产地下有品牌在使用
**解决**：先删除或修改使用该产地的所有品牌，然后再删除产地

### 问题 3：产地选择器不显示任何选项
**原因**：数据库表未创建或未迁移
**解决**：执行数据库迁移脚本（参考第 1 节）

### 问题 4：搜索时产地筛选不起作用
**原因**：可能没有正确选择产地或搜索逻辑有问题
**解决**：
1. 确保在产地下拉框中选择了一个产地（不是"全部产地"）
2. 点击"搜索"按钮
3. 检查浏览器控制台是否有错误
